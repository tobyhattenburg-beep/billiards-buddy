## How the AAB gets built

CI builds the **unsigned** release bundle: `.github/workflows/android-build.yml`
runs `./gradlew bundleRelease assembleRelease` on every push touching `android/**`
(or manually via *Actions → Android build → Run workflow*) and commits the outputs
to `dist/` on the same branch.

Signing happens **offline** with the permanent upload keystore (never in git, never
in CI secrets):

```
jarsigner -keystore signing.keystore -storepass <store-pass> -keypass <key-pass> \
  -sigalg SHA256withRSA -digestalg SHA-256 \
  dist/app-release.aab billiardsbuddy
```

The keystore + passwords live in the owner's off-machine backups (see
`store/OWNER-RUNBOOK.md`). Losing the keystore means the app can never be updated.

## Post-generation patches (re-apply after any `bubblewrap update`)

`bubblewrap update` regenerates `app/` from the template, wiping these manual edits:

1. `app/build.gradle`: `targetSdkVersion 35` → `36` (Play requires 36 for new apps
   from 2026-08-31).
2. `app/src/main/AndroidManifest.xml`: add below POST_NOTIFICATIONS —
   `<uses-permission android:name="android.permission.CAMERA"/>` and
   `<uses-feature android:name="android.hardware.camera" android:required="false"/>`
   (Shot Analyzer).
3. If you regenerated against a locally served manifest, restore the live URL in
   `app/build.gradle` (`webManifestUrl` resValue) and in `twa-manifest.json`
   (`iconUrl`, `maskableIconUrl`, `webManifestUrl` →
   `https://tobyhattenburg-beep.github.io/billiards-buddy/...`).

Note: `bubblewrap update` fetches `webManifestUrl` and the icon URLs at generation
time, so regeneration needs either live-site access or a local server
(`python3 -m http.server` from the repo root, then point those three URLs at
`http://localhost:<port>/...` temporarily).

## Version bumps

Edit `twa-manifest.json` (`appVersion`, `appVersionCode`) **and**
`app/build.gradle` (`versionName`, `versionCode`) together, or re-run
`bubblewrap update` and re-apply the patches above.
